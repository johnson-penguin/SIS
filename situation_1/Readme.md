# 保險業務擴充完成與操作說明 (Walkthrough)

我們已經成功將您所要求的「保險理賠核算」與「健康保險申購」等商業情境，整合進了原有的 SIS 系統中！這次擴充讓這套系統從純粹的「緊急通報系統」，升級為「整合醫療與金融保險服務的綜合平台」。

以下是新增的架構與 Demo 流程說明：

## 新增系統架構
1. **資料庫擴充**：在原本的 SQLite 資料庫 (`iot_data.db`) 中，新增了 `insurance_claims` (理賠單)、`medical_reports` (醫療報告) 與 `insurance_policies` (保單) 三張資料表。
2. **邊緣伺服器 API (`0_edge_server.py`)**：新增了 `/api/insurance/...` 相關的 HTTP POST/GET 介面，作為前端互相交換保險資訊的橋樑。
3. **全新的保險端儀表板 (`4_insurance_ui.py`)**：這是一支全新撰寫的 Flask 程式，運行在 **Port 5004**。它作為保險公司的戰情看版，可以處理理賠、發放保費以及審核保單。

## 🎯 Demo 實作劇本與操作教學

您可以按照以下兩個劇本進行您的 Demo：

### 劇本一：自動化意外理賠與出險結算
**情境**：受保人發生意外 (L4)，住院治療後，由保險公司根據醫院回傳的報告自動撥款結帳。

1. **觸發意外**：在終端模擬器 (`0_single_ue.py`) 按下 `3`，模擬休克或跌倒等 L4 極端危害。
2. **發布住院通知**：
   * 開啟 **管理中心 (`2_web_monitoring_station.py`, Port 5000)**。
   * 您會看到下方跳出「緊急危害觸發」的事件卡片。
   * 卡片內會多出三個按鈕，請點擊 **📝 發布住院通知**。
3. **保險端啟動計數**：
   * 開啟 **保險公司端 (`4_insurance_ui.py`, Port 5004)**。
   * 您會看到「住院理賠與事故處置」區塊出現了一張紅色的理賠單，並且**住院天數與理賠金額正在以每五秒一天的速度快速跳動累積**！
4. **補齊醫療報告**：
   * 回到 **管理中心 (Port 5000)**，依序點擊剛剛卡片上解鎖的 **📄 上傳診療報告** 與 **📅 上傳複診計畫**。
5. **核准與結算出險**：
   * 切換回 **保險公司端 (Port 5004)**。
   * 您會看到理賠單已經收到了這三份報告。此時下方原本反灰的核准按鈕會解鎖。
   * 點擊 **✅ 核准並出院結算**。系統會自動跳出一張精美的 **電子理賠核算單**，展示給客戶看！這張理賠單的狀態也會變成綠色的「已出險結帳」。

### 劇本二：主動提供健康報告申購保險
**情境**：使用者主動打包自己手錶內的健康數據，向保險公司投保；保險公司後續根據健康狀況給予費率減免回饋。

1. **送出報告申請**：
   * 開啟 **使用者端 (`2_web_ue.py`, Port 5001)**。
   * 網頁下方新增了一個「🛡️ 我的保險專區」。
   * 點擊 **📊 匯出長期報告並申購**。
2. **保險端核准保單**：
   * 切換到 **保險公司端 (Port 5004)**。
   * 右側的「申購核險與保單管理」會出現一筆正在「審核中」的新保單。
   * 點擊 **✅ 核准保單**。
3. **給予品質與費率回饋**：
   * 該筆保單會變成生效中，並出現 **💡 發送次年費率調整與建議** 的按鈕。
   * 點擊該按鈕，系統會跳出提示框，您可以自由輸入給客戶的回饋 (例如：*「由於您的健康數據優良，次年保費給予 5% 折扣！」*)。
4. **使用者收到回饋**：
   * 回到 **使用者端 (Port 5001)**，保險專區內的保單狀態已經變成生效中，並且會在下方顯示保險公司剛剛發送的那段「定期回饋訊息」！
